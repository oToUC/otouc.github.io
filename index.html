canvas-图形变换2

图形变换

canvas的图形变换函数是叠加的

  • 记得逆转~translate(100, 100)→translate(-100, -100)→translate(300, 300)
  • 但这样实在是太麻烦了，，
  • 因此save(), restore()出马了～～

Canvas状态的保存与恢复
save()
restore()
当我们绘制一个整体的元素时，特别是用到图形变换时注意使用save()与restore()
来保证canvas图形状态的绘制正确

应用translate, rotate & scale

scale()函数会有副作用，坐标，线条宽度和本身都会scale

变换矩阵
a c e
b d f
0 0 1

1 0 0
0 1 0
0 0 1

a 水平缩放(1)
b 水平倾斜(0)
c 垂直倾斜(0)
d 垂直缩放(1)
e 水平位移(0)
f 垂直位移(0)

a d 缩放
b c 倾斜
e f 位移
transform(a, b, c, d, e, f);
默认
transform(1, 0, 0, 1, 0, 0);

canvas-画弧和圆

{ draw: arc }

画弧其实我们可以很容易联想到以下几点？（弧是圆中的一部分）

  1. 知道圆的中心点(x, y)
  2. 知道圆的半径
  3. 知道弧在圆上的开始于某个弧度
  4. 知道弧在圆上的结束于某个弧度
  5. 是顺时针画，还是逆时针画
  • 因此在canvas中的对应的arc方法参数也就不言而喻了
  • 实际演练
  1. beginPath()指的是重新规划一条路径
  2. closePath()指的是要结束当前的路径，如果当前的路径没有封闭的话，会自动让当前的路径封闭
  3. beginPath() 与 closePath()不一定要成对出现
  4. 对于context.fill(), closePath()写与不写都是先把路径封闭，然后再填充
